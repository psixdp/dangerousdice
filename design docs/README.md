# 产品需求文档规范

本目录下的设计文档遵循以下统一规范。

## 文档结构

- **main.md**：总览与主流程（概述、流程机制）
- **game.md**：对局与关卡规则
- **dice.md**：骰子、升级与消耗品
- **ui.md**：界面与交互（含结算界面）

## 格式规范

### 标题
- 文档标题：一级标题 `# 文档标题`，且与文件名主题一致。
- 章：二级标题 `## N 章节名`，N 从 1 开始连续编号。
- 节：三级标题 `### N.M 节名`，N 为章号，M 从 1 开始。
- 编号与标题之间保留一个空格，如 `## 1 概述`、`### 2.1 准备`。

### 编号规则
- 编号从 1 开始，不继承其他文档（如 ui 用 1、2、3，不用 4.1、4.2）。
- 层级：章 `N` → 节 `N.M` → 子节 `N.M.K`（若需要）。

### 正文与列表
- 章/节下方说明用段落；子项用 Markdown 列表 `-`。
- 子项需要字母编号时使用：`- a. 内容`、`- b. 内容`。
- 不采用纯空格缩进表示层级，用标题层级和列表表示。

### 其他
- 数学/公式中的乘号使用 `×` 或 `*`，必要时用粗体：**基础分 + 剩余次数 × 2**。
- 待补充内容用块引用注明：`> 待补充`，不使用 HTML 注释。
