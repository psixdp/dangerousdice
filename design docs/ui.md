# 界面设计 · Figma 规格

画布统一：**1280 × 720 px**，固定尺寸，无滚动。所有界面在此画布内完成布局。

---

## 界面 1：主菜单（Main Menu）

**用途**：选择对局类型、选择初始骰子、进入游戏。

### 布局区域
| 区域 | 建议范围 | 说明 |
|------|----------|------|
| 顶部标题区 | 顶部 0–120px | 游戏主标题，视觉焦点 |
| 中央选择区 | 垂直居中约 120–520px | 对局选择 + 骰子选择，左右或上下分组 |
| 底部操作区 | 底部 520–720px | 主按钮、帮助入口 |

### 组件清单
| 类型 | 文案/标签 | 位置 | 备注 |
|------|------------|------|------|
| 标题 | 千王之王 | 顶部标题区居中 | 大号主标题，可配副标题如「骰子 roguelite」 |
| 标签 | 选择对局 | 中央选择区左侧或上方 | 分组标题 |
| 选择器/卡片组 | （对局选项） | 中央选择区 | 展示「经典对局」「贪吃蛇」等，当前选中高亮；每项可带简短说明（如初始积分） |
| 标签 | 选择初始骰子 | 中央选择区 | 分组标题 |
| 选择器/卡片组 | （骰子选项） | 中央选择区 | 展示可选骰子类型（标准六面/八面/灌铅等），单选，选中 1 种 |
| 主按钮 | 开始游戏 | 底部操作区居中偏下 | 主 CTA，大而醒目 |
| 次要按钮/链接 | 游戏说明 或 ？ | 底部角落（如右下） | 点击后弹出或跳转帮助内容，不默认展开 |

### 层级与间距
- 主标题 > 开始游戏按钮 > 两组选择区 > 游戏说明。
- 选择区内部留足间距，便于点选；整体保持垂直居中感。

## 界面 2：关卡界面（Level / Game）

**用途**：投掷骰子、查看累计分与历史、使用消耗品；返回即放弃本局。

### 布局区域
| 区域 | 建议范围 | 说明 |
|------|----------|------|
| 顶栏 | 顶部 0–56px | 返回按钮 + 本关信息（总次数、剩余次数、目标点数） |
| 左侧信息区 | 左侧 0–280px 或 顶部信息下方 | 投掷历史列表 + 消耗品使用历史，可折叠/展开 |
| 中央游戏区 | 画布中部 | 骰子桌 + 当前拥有的骰子展示（视觉焦点） |
| 右侧或底部操作区 | 右侧或底部 620–720px | 投掷按钮、消耗品背包 |

### 组件清单
| 类型 | 文案/标签 | 位置 | 备注 |
|------|------------|------|------|
| 按钮 | 返回主菜单 | 顶栏左侧 | 次要样式；点击放弃本局，需二次确认或直接返回 |
| 文本 | 第 N 关、目标：XX 点 | 顶栏 | 本关目标点数 |
| 文本 | 总投掷次数：X / 剩余：Y | 顶栏或顶栏下 | 本关总次数与剩余次数 |
| 文本 | 当前累计：Z | 顶栏或信息区顶部 | 本关累计总分 |
| 标题/标签 | 投掷历史 | 左侧信息区 | 列表标题 |
| 列表项 | 第 1 次：XX 分 [可展开] | 左侧信息区 | 默认只显示每次总分；点击展开显示该次各骰子点数 |
| 标题/标签 | 消耗品使用 | 左侧信息区（可选） | 有使用时显示；列表记录使用的消耗品及时机 |
| 骰子展示区 | （骰子视觉） | 中央游戏区 | 骰子桌 + 当前拥有的骰子，可多颗并排 |
| 主按钮 | 投掷 | 操作区 | 主 CTA；点击后展示本次结果与当前累计积分 |
| 背包/格子 | 消耗品槽位 ×2 | 操作区 | 显示已装备的消耗品，点击使用（仅关卡内、非投掷中可用） |

### 交互与状态
- 投掷后：在界面中反馈本次点数与累计分（弹窗或固定结果区）。
- 投掷历史：列表可滚动；单条可展开/收起明细。
- 返回主菜单：可弹确认「放弃本局？」再返回。

## 界面 3：出千商店（Shop）

**用途**：用积分购买骰子升级、消耗品、新骰子；可跳过直接进入下一关；返回即放弃本局。

### 布局区域
| 区域 | 建议范围 | 说明 |
|------|----------|------|
| 顶栏 | 顶部 0–56px | 返回按钮 + 当前积分 |
| 商品区 | 中部 56–520px | 4 个槽位：2 个升级 + 1 个消耗品 + 1 个新骰子 |
| 下一关信息区 | 商品区下方或右侧 | 下一关目标点数等 |
| 底部操作区 | 底部 620–720px | 刷新商店、进入下一关 |

### 组件清单
| 类型 | 文案/标签 | 位置 | 备注 |
|------|------------|------|------|
| 按钮 | 返回主菜单 | 顶栏左侧 | 同关卡界面，放弃本局 |
| 文本 | 积分：XXX | 顶栏右侧 | 当前持有积分 |
| 卡片/槽位 | 骰子升级 1 | 商品区 | 展示升级类型与价格，点击选择骰子并购买 |
| 卡片/槽位 | 骰子升级 2 | 商品区 | 同上，与升级 1 不重复类型 |
| 卡片/槽位 | 消耗品 | 商品区 | 展示消耗品类型与价格，购买后入背包 |
| 卡片/槽位 | 新骰子 | 商品区 | 展示骰子类型与价格，仅从「额外骰子」池随机 |
| 按钮 | 刷新商店 ( -X 积分 ) | 商品区旁或底部 | 消耗积分刷新，成本由配置决定 |
| 文本 | 下一关：目标 XX 点 | 下一关信息区 | 下一关目标点数 |
| 主按钮 | 进入下一关 | 底部操作区 | 无需购买也可点击，进入下一关 |

### 层级与间距
- 当前积分与「进入下一关」突出；商品卡片等大、易点；刷新为次要操作。

## 界面 4：结算界面（Victory）

**用途**：通关最后一关后展示胜利；当前为简单版，后续可扩展。

### 布局区域
| 区域 | 建议范围 | 说明 |
|------|----------|------|
| 全屏覆盖 | 1280×720 | 半透明遮罩 + 居中内容框 |

### 组件清单
| 类型 | 文案/标签 | 位置 | 备注 |
|------|------------|------|------|
| 标题 | You Win 或 胜利 | 画布中央 | 大号，庆祝感 |
| 副标题（可选） | 恭喜通关！ | 标题下方 | 小一号 |
| 主按钮 | 返回主菜单 | 标题下方 | 点击回到主菜单 |

### 扩展预留
- 后续可在此增加：本局总积分、解锁内容、成就等；布局预留中央偏下区域放按钮与列表。

## 通用规范（供 Figma 参考）

- **字体**：主标题建议 36–48px；正文/列表 14–18px；按钮文字 16–20px。
- **主按钮**：高对比度、圆角；主菜单与关卡/商店的 CTA 统一风格。
- **次要按钮**：如返回、刷新、帮助，弱一档，不抢主 CTA。
- **卡片/槽位**：统一圆角与内边距；选中态有明显边框或背景变化。
- **列表**：行高舒适，可展开行有箭头或「展开」提示。
- **颜色**：可设主色（如深色底 + 金色/琥珀强调）以区分千王/赌场主题；危险操作（如放弃本局）可用次要色或描边区分。
